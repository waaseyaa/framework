# Waaseyaa Roadmap

Issue tracking and milestone planning is managed on GitHub:

- **Milestones:** https://github.com/waaseyaa/waaseyaa/milestones
- **Open issues:** https://github.com/waaseyaa/waaseyaa/issues

## Milestones

| Milestone | Description |
|---|---|
| [v0.1 — Framework Identity](https://github.com/waaseyaa/waaseyaa/milestone/1) | Kernel, skeleton, config-driven entity types, CLI, publishing automation |
| [v0.2 — Auth & Storage](https://github.com/waaseyaa/waaseyaa/milestone/2) | JWT/API key auth, disk-backed file storage, file upload, dev server stability |
| [v0.3 — Native SSR Engine](https://github.com/waaseyaa/waaseyaa/milestone/4) | Twig rendering pipeline, field formatters, view modes, public routing, theme system |
| [v0.4 — AI & MCP](https://github.com/waaseyaa/waaseyaa/milestone/5) | Vector embeddings, semantic search, MCP endpoint — AI layer online before the content model |
| [v0.5 — System Integration & Verification](https://github.com/waaseyaa/waaseyaa/issues/57) | End-to-end verification in the new Minoo app: Playwright E2E, SSR snapshots, semantic search, MCP integration |
| [v0.6 — Minoo Content Model & Authoring Foundation](https://github.com/waaseyaa/waaseyaa/issues/73) | Relationship-first domain model, deterministic fixtures, SSR/search/MCP traversal contracts, and v0.6 release gate |
| [v1.0 — Platform Hardening & API Stabilization](https://github.com/waaseyaa/waaseyaa/issues/106) | Stable traversal/SSR/MCP/semantic contracts, security hardening, deterministic release verification |
| [v1.1 — Performance, Caching, and Scale](https://github.com/waaseyaa/waaseyaa/issues/125) | SSR and MCP read-path caching, semantic refresh operations, hot-path optimization, regression baselines |
| [v1.2 — Developer Ergonomics, Tooling, and Ecosystem Readiness](https://github.com/waaseyaa/waaseyaa/issues/134) | Deterministic scaffolding, fixture generation, debug/perf diagnostics, MCP introspection, onboarding consolidation |
