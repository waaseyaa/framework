## Unit 2 — #347: Metapackage smoke tests

**Branch:** `fix/347-metapackage-smoke`

**Files:**
- Create: `tests/Integration/Phase23/MetapackageSmokeTest.php`

### Background

The three metapackages are pure Composer dependency declarations with no PHP source code.
Their value is ensuring the right packages are required together. A smoke test validates that:
1. Key classes from each declared dependency actually exist in the autoloader
2. No class-loading conflicts prevent the set from booting together

Since this is a monorepo with path repositories, all dependencies are always installed.
The tests assert `class_exists()` for one representative class per package in the dependency
tree.

### Representative classes to probe

| Metapackage | Dependency | Probe class |
|---|---|---|
| core | waaseyaa/foundation | `Waaseyaa\Foundation\Kernel\AbstractKernel` |
| core | waaseyaa/entity | `Waaseyaa\Entity\EntityType` |
| core | waaseyaa/access | `Waaseyaa\Access\Gate\AccessResult` |
| core | waaseyaa/user | `Waaseyaa\User\User` |
| core | waaseyaa/routing | `Waaseyaa\Routing\RouteBuilder` |
| cms | waaseyaa/node | `Waaseyaa\Node\Node` |
| cms | waaseyaa/taxonomy | `Waaseyaa\Taxonomy\TaxonomyTerm` |
| cms | waaseyaa/media | `Waaseyaa\Media\Media` |
| cms | waaseyaa/api | `Waaseyaa\Api\JsonApiController` |
| full | waaseyaa/ai-schema | `Waaseyaa\AI\Schema\AiSchemaServiceProvider` |
| full | waaseyaa/mcp | `Waaseyaa\Mcp\McpServiceProvider` |

Before writing the test, verify the exact class names exist by running:
```bash
grep -r 'class AbstractKernel' packages/foundation/src/ | head -1
grep -r 'class EntityType ' packages/entity/src/ | head -1
grep -r 'class AccessResult' packages/access/src/ | head -1
grep -r 'class Node ' packages/node/src/ | head -1
grep -r 'class TaxonomyTerm' packages/taxonomy/src/ | head -1
grep -r 'class Media ' packages/media/src/ | head -1
grep -r 'class AiSchemaServiceProvider\|class SchemaServiceProvider' packages/ai-schema/src/ | head -2
grep -r 'class McpServiceProvider\|class MCP' packages/mcp/src/ | head -2
```
Adjust the probe classes in the test based on what actually exists.

### Steps

- [ ] Run the grep commands above to confirm exact class names. Adjust the table above if needed.

- [ ] Create `tests/Integration/Phase23/MetapackageSmokeTest.php`:
  ```php
  <?php

  declare(strict_types=1);

  namespace Waaseyaa\Tests\Integration\Phase23;

  use PHPUnit\Framework\Attributes\CoversNothing;
  use PHPUnit\Framework\Attributes\DataProvider;
  use PHPUnit\Framework\Attributes\Test;
  use PHPUnit\Framework\TestCase;

  /**
   * Smoke tests for metapackages: cms, core, full.
   *
   * These tests verify that one representative class from each declared
   * dependency is autoloadable. They catch broken autoloader config,
   * missing package splits, or namespace regressions.
   */
  #[CoversNothing]
  final class MetapackageSmokeTest extends TestCase
  {
      /** @return array<string, array{string}> */
      public static function coreClasses(): array
      {
          return [
              'foundation kernel'   => [\Waaseyaa\Foundation\Kernel\AbstractKernel::class],
              'entity type'         => [\Waaseyaa\Entity\EntityType::class],
              'access result'       => [\Waaseyaa\Access\Gate\AccessResult::class],
              'user'                => [\Waaseyaa\User\User::class],
              'route builder'       => [\Waaseyaa\Routing\RouteBuilder::class],
          ];
      }

      /** @return array<string, array{string}> */
      public static function cmsClasses(): array
      {
          return [
              'node'                => [\Waaseyaa\Node\Node::class],
              'taxonomy term'       => [\Waaseyaa\Taxonomy\TaxonomyTerm::class],
              'media'               => [\Waaseyaa\Media\Media::class],
              'json api controller' => [\Waaseyaa\Api\JsonApiController::class],
          ];
      }

      /** @return array<string, array{string}> */
      public static function fullClasses(): array
      {
          return [
              // Adjust these based on actual class names found in the grep step
              'ai schema provider'  => [\Waaseyaa\AI\Schema\AiSchemaServiceProvider::class],
              'mcp provider'        => [\Waaseyaa\Mcp\McpServiceProvider::class],
          ];
      }

      #[Test]
      #[DataProvider('coreClasses')]
      public function core_metapackage_dependencies_are_autoloadable(string $class): void
      {
          $this->assertTrue(
              class_exists($class) || interface_exists($class) || abstract_class_exists_hack($class),
              "Class {$class} (waaseyaa/core dependency) not found in autoloader."
          );
      }

      #[Test]
      #[DataProvider('cmsClasses')]
      public function cms_metapackage_dependencies_are_autoloadable(string $class): void
      {
          $this->assertTrue(
              class_exists($class),
              "Class {$class} (waaseyaa/cms dependency) not found in autoloader."
          );
      }

      #[Test]
      #[DataProvider('fullClasses')]
      public function full_metapackage_dependencies_are_autoloadable(string $class): void
      {
          $this->assertTrue(
              class_exists($class),
              "Class {$class} (waaseyaa/full dependency) not found in autoloader."
          );
      }
  }
  ```

  **Note on `abstract_class_exists_hack`:** Don't use it — `class_exists()` returns true for
  abstract classes. Remove that call and use only `class_exists($class)` for all three methods.
  Also fix the `fullClasses` probe classes based on what the grep step returns.

  **Clean version of the three test methods (use this):**
  ```php
  #[Test]
  #[DataProvider('coreClasses')]
  public function core_metapackage_dependencies_are_autoloadable(string $class): void
  {
      $this->assertTrue(
          class_exists($class),
          "Class {$class} (waaseyaa/core dependency) not found in autoloader."
      );
  }
  ```
