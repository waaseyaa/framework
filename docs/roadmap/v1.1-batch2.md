## Research Summary

**#339 (version string):**
- `packages/foundation/src/Kernel/ConsoleKernel.php:176` has `'version' => '0.1.0'` hardcoded
- Root `composer.json` has no `"version"` field
- Best canonical source: add `"version": "1.1.0"` to root `composer.json`, read it via
  `InstalledVersions::getRootPackage()['version']` (Composer runtime API, always available)
- Fallback constant in `Foundation\Version` class if Composer runtime not available

**#343 (favicon):**
- Nuxt 3 serves static assets from `packages/admin/public/` (relative to project root, not srcDir)
- `public/` dir does not exist yet ŌĆö needs to be created
- A minimal 1├Ś1 transparent ICO is sufficient; no design required
- PHP backend (`public/index.php`) is API-only and doesn't need a favicon

**#341 (redundant proxy):**
- `nuxt.config.ts` has both `nitro.devProxy` (dev-only) AND `routeRules` (all modes) for `/api`
- `routeRules` with `proxy:` is the correct Nitro mechanism for all environments
- Fix: remove `nitro.devProxy` block entirely; keep `routeRules`
- No behavior change ŌĆö `routeRules` already handles dev mode

**#335 (nav group labels):**
- `useNavGroups.ts` generates `labelKey: \`nav_group_\${key}\`` for every group
- `NavBuilder.vue:35` renders `{{ t(group.labelKey) }}`
- `en.json` has keys for the 13 known groups (people, content, taxonomy, etc.)
- Unknown groups (custom entity types with `group: 'communities'`) return the raw key string
- Fix: add `label` property to `ResolvedNavGroup` with a humanized fallback, use
  `t(group.labelKey, group.label)` in NavBuilder ŌĆö vue-i18n uses second arg as default

  const ico = Buffer.from([
    0x00,0x00, 0x01,0x00, 0x01,0x00, // ICO header: reserved, type=1, count=1
    0x01,0x01,                         // image: width=1, height=1
    0x00,0x00,                         // colorCount=0, reserved=0
    0x01,0x00, 0x01,0x00,             // planes=1, bitCount=1
    0x30,0x00,0x00,0x00,              // sizeInBytes=48
    0x16,0x00,0x00,0x00,              // offset=22
    // BITMAPINFOHEADER (40 bytes)

    0x01,0x00,0x00,0x00,              // biWidth=1
    0x02,0x00,0x00,0x00,              // biHeight=2 (1*2 for XOR+AND)
    0x01,0x00,                         // biPlanes=1
    0x01,0x00,                         // biBitCount=1
    0x00,0x00,0x00,0x00,              // biCompression=0
    0x00,0x00,0x00,0x00,              // biSizeImage=0
    0x00,0x00,0x00,0x00,              // biXPelsPerMeter=0
    0x00,0x00,0x00,0x00,              // biYPelsPerMeter=0
    0x00,0x00,0x00,0x00,              // biClrUsed=0
    0x00,0x00,0x00,0x00,              // biClrImportant=0
    // Color table (2 colors * 4 bytes)
    0x00,0x00,0x00,0x00,              // color 0: transparent black
    0xFF,0xFF,0xFF,0x00,              // color 1: white
    // XOR mask (1 byte padded to 4)
    0x00,0x00,0x00,0x00,
    // AND mask (1 byte padded to 4) ŌĆö all transparent
    0x80,0x00,0x00,0x00,
