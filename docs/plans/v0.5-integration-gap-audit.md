# v0.5 Integration Gap Audit

Date: 2026-03-04  
Scope: Waaseyaa v0.1-v0.4 validated through Minoo v0.5 verification

## Summary

Confirmed integration gaps discovered during v0.5 verification:

| Gap | Area | Priority | Status | Tracking |
| --- | --- | --- | --- | --- |
| Dev fallback account is gated by SAPI, not environment/config | Middleware/auth | P1 | Open | [#69](https://github.com/waaseyaa/waaseyaa/issues/69) |
| Public SSR flow does not wire language negotiation into alias resolution/cache context | SSR/locale | P1 | Open | [#70](https://github.com/waaseyaa/waaseyaa/issues/70) |
| Embeddings are not cleaned up on entity delete | AI lifecycle | P2 | Open | [#71](https://github.com/waaseyaa/waaseyaa/issues/71) |
| Semantic search does not degrade gracefully on provider failure/dimension drift | AI/provider parity | P1 | Open | [#72](https://github.com/waaseyaa/waaseyaa/issues/72) |

## Findings

### 1) Middleware/Auth: dev fallback guard is too broad (P1)

- Evidence:
  - `packages/foundation/src/Kernel/HttpKernel.php` wires `SessionMiddleware` with `new DevAdminAccount()` when `PHP_SAPI === 'cli-server'`.
  - `packages/user/src/DevAdminAccount.php` has all-permissions behavior.
- Risk:
  - Auth behavior is tied to runtime SAPI rather than explicit environment/config intent.
- Issue: [#69](https://github.com/waaseyaa/waaseyaa/issues/69)

### 2) SSR Locale/Alias integration gap (P1)

- Evidence:
  - `packages/foundation/src/Kernel/HttpKernel.php` `handleRenderPage()` calls `PathAliasResolver::resolve($normalizedPath)` without negotiated langcode.
  - `packages/path/src/PathAliasResolver.php` defaults `langcode` to `'en'`.
  - `packages/routing/src/Language/LanguageNegotiator.php` exists but is not wired into `HttpKernel`.
- Risk:
  - Locale-specific aliases and SSR rendering are not driven by negotiated/requested language context.
- Issue: [#70](https://github.com/waaseyaa/waaseyaa/issues/70)

### 3) Embedding lifecycle deletion gap (P2)

- Evidence:
  - `packages/ai-vector/src/EntityEmbeddingListener.php` only implements `onPostSave`.
  - `packages/ai-vector/src/SqliteEmbeddingStorage.php` supports `store`/`findSimilar` but no deletion API.
- Risk:
  - Orphaned vectors accumulate after content deletion.
- Issue: [#71](https://github.com/waaseyaa/waaseyaa/issues/71)

### 4) Provider resilience/fallback gap (P1)

- Evidence:
  - `packages/ai-vector/src/SearchController.php` semantic path returns empty on provider errors rather than degrading to keyword mode.
  - `packages/ai-vector/src/SqliteEmbeddingStorage.php` silently skips dimension mismatches.
- Risk:
  - Provider outages or provider/model changes can produce silent empty semantic results.
- Issue: [#72](https://github.com/waaseyaa/waaseyaa/issues/72)
