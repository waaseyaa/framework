# v1.1 SSR Cache Key Hardening Report

Date: 2026-03-04  
Scope: `#126 v1.1: SSR cache key hardening (workflow + graph + locale context)`

## Delivered

- Hardened SSR cache variant builder in `HttpKernel` to include deterministic dimensions:
  - locale/langcode
  - route view mode
  - preview/public mode
  - workflow state
  - graph-context hash
  - contract version payload hash
- Introduced v1.1 variant key format:
  - `v2:{langcode}:{view_mode}:{public|preview}:{workflow_state}:{hash}`
- Added token sanitization for human-readable key segments.

## Security/Isolation Outcome

- Public and preview render variants are now explicitly partitioned at key level.
- Public cache read/write gates remain enforced for unauthenticated non-preview traffic.
- Added integration proof that preview requests do not use persisted render cache.
