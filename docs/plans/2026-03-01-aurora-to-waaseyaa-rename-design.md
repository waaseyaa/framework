# Rename: Aurora → Waaseyaa

## Decision
The project "Aurora CMS" is being renamed to "Waaseyaa" to avoid conflict with an existing Aurora CMS product.

## Scope

### In scope
- All PHP namespaces (`Aurora\` → `Waaseyaa\`)
- Composer package names (`aurora/*` → `waaseyaa/*`)
- 9 PHP class files with "Aurora" in the filename
- CLI binary (`bin/aurora` → `bin/waaseyaa`)
- Environment variables (`AURORA_DB` → `WAASEYAA_DB`, `AURORA_CONFIG_DIR` → `WAASEYAA_CONFIG_DIR`)
- Default database filename (`aurora.sqlite` → `waaseyaa.sqlite`)
- Display strings: "Aurora CMS" → "Waaseyaa" (no "CMS" suffix)
- README.md and CLAUDE.md

### Out of scope
- Historical design docs in `docs/plans/` (left as-is)
- Vendor directories (auto-regenerated by Composer)

## Replacements

| Find | Replace | Context |
|------|---------|---------|
| `Aurora\` | `Waaseyaa\` | PHP namespaces, use statements |
| `aurora/` | `waaseyaa/` | Composer package names |
| `Aurora CMS` | `Waaseyaa` | Display strings |
| `AURORA_DB` | `WAASEYAA_DB` | Environment variable |
| `AURORA_CONFIG_DIR` | `WAASEYAA_CONFIG_DIR` | Environment variable |
| `aurora.sqlite` | `waaseyaa.sqlite` | Default database filename |
| `bin/aurora` | `bin/waaseyaa` | CLI entry point |

- `packages/cli/tests/Unit/AuroraApplicationTest.php` → `WaaseyaaApplicationTest.php`
- `packages/testing/src/AuroraTestCase.php` → `WaaseyaaTestCase.php`
- `packages/testing/tests/Unit/AuroraTestCaseTest.php` → `WaaseyaaTestCaseTest.php`
- `packages/plugin/src/Attribute/AuroraPlugin.php` → `WaaseyaaPlugin.php`
- `packages/routing/src/AuroraRouter.php` → `WaaseyaaRouter.php`
- `packages/routing/tests/Unit/AuroraRouterTest.php` → `WaaseyaaRouterTest.php`

## Approach
Scripted bulk rename using `sed`/`find`, then verify with full test suite.
