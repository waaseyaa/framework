  - file: `packages/foundation/src/Cache/DiscoveryCachePrimitives.php`
- Refactored `HttpKernel` discovery cache behavior to use shared primitives for:
  - deterministic cache key generation,
  - contract metadata envelope,
  - cache tag derivation.

## Invalidation Hardening

- Expanded discovery cache tagging to include related entities present in cached payloads:
  - topic/timeline `items[*].related_entity_type/id`,
  - cluster `related_entities[*].type/id`,
  - endpoint browse edges (`browse.outbound` / `browse.inbound`).
- This closes stale-cache risk when related entities change but source entity does not.

## Determinism and Stability

- Key generation remains deterministic under equivalent option ordering.
- Payload contract metadata remains stable (`v1.0`, `stable`, `discovery_api` default).
- Existing anonymous hit/miss behavior is preserved while primitives are now reusable.
